# SereneMind - Student Mental Wellness Tracker

A complete, production-ready, and responsive Mental Wellness Tracker web application designed specifically for students. It offers a beautiful, calming light/dark theme with pastels, emerald greens, and soft sky blues.

## Features Included

1. **User Authentication**: Secure Sign-up, Log-in, and profile fetching powered by JWT and bcrypt password hashing.
2. **Daily Mood Tracker**: Animated emoji logging (1-5 scale) with custom tags and journal linkages.
3. **Private Journal**: Markdown-ready workspace with full-text search and date range filters.
4. **AI Reflection Assistant**: Reflective summaries, cognitive patterns, and micro-tips generated by OpenAI's `gpt-4o-mini` (equipped with a robust local mock mode and crisis keyword checks that link to distress hotlines).
5. **Pomodoro Study Timer**: A 25/5 focus interval timer with circular progress animation, alert sound, and custom task list.
6. **Breathing Exercise Widget**: Interactive breathing cycles (Box breathing and 4-7-8 method) with smooth SVG scale expand/contract animations.
7. **Mood Analytics Dashboard**: Beautiful charts (line chart for trends, bar chart for tag correlations, pie chart for score distributions) built with Recharts.
8. **Student Wellness Insights**: Dynamically calculated wellness articles and guidelines personalized to the student's logged mood.
9. **Responsive Design**: Mobile-first layout with smooth theme toggling.

## Local Setup Instructions

### Prerequisites

- **Node.js** (v18.0.0 or higher recommended)
- **npm** (v9.0.0 or higher)
- **MongoDB** (A running local MongoDB instance or a free MongoDB Atlas connection string)

### 1. Install Dependencies

You can install all dependencies for both the frontend and backend in one command from the project root:

```bash
npm run install-all
```

### 2. Configure Environment Variables

#### Backend Configuration

Navigate to the `backend/` directory, copy the `.env.example` file to `.env`, and fill in your variables:

```env
PORT=5000
NODE_ENV=development
CORS_ORIGIN=http://localhost:3000

# MongoDB URI (Replace with MongoDB Atlas connection string)
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.mongodb.net/mental-wellness?retryWrites=true&w=majority

# Authentication
JWT_SECRET=super_secret_jwt_sign_key_change_me_in_production

# OpenAI API Key (Required for live reflections, fallback dummy response if empty)
OPENAI_API_KEY=your_openai_api_key_here
```

#### Frontend Configuration

Navigate to the `frontend/` directory, create a `.env` file (or update variables if present):

```env
VITE_API_URL=http://localhost:5000/api
```

### 3. Seed Mock Data

To pre-populate the database with a test user (`test@wellness.edu` / `password123`) and a history of journal entries/mood logs for charts:

```bash
npm run seed
```

### 4. Start Local Development Servers

Run the following command from the project root. It will run both the Node.js backend (on port `5000`) and the Vite React frontend (on port `3000` / next available port) concurrently:

```bash
npm run dev
```

Open your browser and navigate to the frontend URL printed in the console (typically `http://localhost:5173` or similar).

## Seeding & Testing Without Live MongoDB

If you want to run the project but do not have MongoDB running locally, you can create a free cluster on [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) in 3 minutes:
1. Register for a free account.
2. Build a Database (choose M0 Free cluster).
3. Under Security -> Network Access, add IP Address `0.0.0.0/0` (allow access from anywhere).
4. Under Database Access, create a database user and copy the credentials.
5. Click **Connect** -> **Drivers**, and copy the connection string.
6. Replace `<username>` and `<password>` in the connection string and add it to your `backend/.env` under `MONGODB_URI`.

## Production Deployment Guide

### Backend Deployment (Render)

1. Register or Log in to [Render](https://render.com/).
2. Create a new **Web Service** and connect your GitHub repository.
3. Configure the following service settings:
   - **Environment**: `Node`
   - **Root Directory**: `backend`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
4. Add your Environment Variables in the Render dashboard settings:
   - `MONGODB_URI`: *Your MongoDB Atlas Connection String*
   - `JWT_SECRET`: *A secure random string*
   - `OPENAI_API_KEY`: *Your OpenAI API Key*
   - `CORS_ORIGIN`: *The URL of your deployed frontend (e.g. `https://your-app.vercel.app`)*
   - `NODE_ENV`: `production`

### Frontend Deployment (Vercel)

1. Register or Log in to [Vercel](https://vercel.com/).
2. Click **Add New** -> **Project** and select your GitHub repository.
3. In the project setup panel:
   - Set **Framework Preset** to `Vite`.
   - Set **Root Directory** to `frontend`.
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. In the Environment Variables section, add:
   - `VITE_API_URL`: *The URL of your deployed Render service API (e.g., `https://serene-mind-api.onrender.com/api`)*
5. Click **Deploy**. Vercel will build and host your frontend.

## Safety and Ethics Safeguard

SereneMind includes a backend text scanner (`backend/controllers/aiController.js`) that detects indicators of crisis or distress (e.g., keywords like `suicide`, `self-harm`, etc.). If triggered, the AI assistant bypasses normal reflection generation and immediate displays confidential crisis hotlines (e.g., `988` Suicide & Crisis Lifeline) to protect the student.
